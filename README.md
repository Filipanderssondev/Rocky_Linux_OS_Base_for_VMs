Choice between .qcow2 and .iso based on requirements:

| Feature                  | `.qcow2` Cloud Image            | Minimal `.iso` Installer |
| ------------------------ | ------------------------------- | ------------------------ |
| **Setup style**          | Automated (cloud-init)          | Manual installer         |
| **Best for**             | Templates, cloning, fast deploy | One-off installations    |
| **Networking**           | Configured via cloud-init       | Set during install       |
| **Partitioning**         | Fixed, simple                   | Fully customizable       |
| **Proxmox template use** | Excellent                       | Poor                     |
| **Effort**               | Very low                        | Moderate                 |

1. Download and verify Rocky Linux

Download the Generic Cloud version (.qcow2) of Rocky Linux (currently v10.1) from the official site (https://rockylinux.org/download). Also download the CHECKSUM file.

Open a terminal and go to the download location: cd ./Downloads

Compute the hash with:
sha256sum Rocky-10-GenericCloud-Base.latest.x86_64.qcow2

Confirm with: sha256sum -c CHECKSUM | grep OK

---------------------------------------------------------------------------------------------------------

---------------------------------------------------------------------------------------------------------


2. Create a Rocky Linux VM

2.1 Create a new VM (Blue button in top-right corner)

settings used:

general
	Name: rocky-base
	Add to HA: No
	Start at boot: No
	
OS: 
	Type: Linux
	Version: 6.x - 2.6 Kernel
	Do not use any media
	
System:
	Graphic Card: Default
	Machine: q35
	BIOS: OVMF (UEFI)
	Add EFI Disk: Yes
	EFI Storage: local-lvm
	Pre-Enroll keys: yes
	SCSI ontroller: VirtiO SCSI single
	
Disks:
	No Disks

CPU:
	Sockets: 1
	Cores: 3
	Type: host

The system uses an Intel Core i7-12700H, with a total of 14 cores. Assigning 3 cores to each VM seems like a good balance. There is still room to spare, and there is no contention. 

Memory:
	Memory (MiB): 12288
	Minimum memory (MiB): 4096
	Ballooning Device: Yes
	Allow KSM: Yes

With 64GB RAM available on this system, we can allocate a decent chunk to each VM. 12GiB of RAM should be plenty for a single Rocky Linux VM. 

Network:
	Bridge: vmbr0
	Model: VirtIO (paravirtualized)
	Vlan Tag: No?
	MAC address: Use the autogenerated address
	Firewall: No


2.2 Go to the Datacenter view > Storage > select local > Add 'Import' to Content list

2.3 In Local > Go to Import > Upload > Select File > Choose the cloud image file. 

2.4 Go to the new VM > Hardware > Add > Import Hard Disk
Important Storage: local
Select Image: Rocky-10-GenericCloud-Base.latest.x86_64.qcow2
Target Storage: local-lvm

Also Add > CloudInit Drive

2.5 Go to the Cloud-Init menu for the VM
Choose a username and password

2.6 Configure Ip addresses for VM
Go to Cloud-Init and then IP Config (net0).
We set a static ip address for our VM. 
If you have access to a DHCP server, that would work just as well.

2.7 Go to the Options menu for the VM
Change the boot order:
1. scsi0
2. ide2
3. net0

2.8 Start the VM and log in
